GOLDi.Core.ComponentModel Namespace

 
Classes
  ClassDescription
ComponentAttribute
Marks a class as a compontent. Each component class will be automatically instantiated, if it was an exported (public) type and the containing assembly was found during the plugin resolution process.
ComponentLoader
This class performs the dependency resolution process and object creation.
LifecycleManager
This class implements the object lifecycle for the component model.
OptionalAttribute
If applied to a parameter, it declares a parameter as optional during dependency resolution. If no value at all, or more than one valuefor the type can be found, the parameter will be set to the default value of the type. When applied to a whole class, failed parameter substitution will not cause an error.
Interfaces
  InterfaceDescription
IAsyncStartable
Signals that an object must be started after it was created. This interface should be used if the start action is asynchronous. For synchronous start actions, consider IStartable.
IAsyncStoppable
Singals that an object must be stopped for a normal shutdown. This interface supplements IDisposable and should only be used, if the shutdown action has to perform asynchronous calls.
IBindingPolicy
Creates a customization point in the dependency resolution process.
ICircularDependencyT
This auxilliary interface can be used in components which have circular references to other components. Instead of using constructor parameters, one class has to implement the ICircularDependencyT interface with the type of other component. After all objects have been created, the resolve function will be called.
ICollectionDependencyT
This auxilliary interface can be used for components that require a complete list of all other components of a specific type. Collection dependency resolution is performed as the last step.
IComponentInstantiationFilter
Defines an interface for checking whether an object of a specific class can/has to be created. This interface should be implemented by attributes which are given to the type.
IFactoryDependency
If a component requires support for the dynamic creation of additional objects using the dependency binding mechanisms used to create itself, it can implement this interface. After the resolution process is completed, an object implementing the IGenericFactory-Interface will be created and passed to the ResolveDependency(IGenericFactory) method.
IGenericFactory
Defines an interface that allows the creation of objects of arbitrary types. During the creation, the dependency resolution mechanisms of the component model will be used.
IStartable
Signals that an object must be started after it was created. This interface should be used if the start action is synchronous. For asynchrnous start actions, consider IAsyncStartable.
